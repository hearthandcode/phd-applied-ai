### [~11:30 EDT] Tool-Initiation Fatigue During Agent Migration — Context Persistence Failure and Conception-to-Execution Boundary Breakdown

**Type:** behavioral-affective
**RQ:** RQ5
**Tags:** executive-function · task-initiation · conception-execution-gap · cognitive-load · context-failure · tool-friction · rejection-sensitivity

**Session overview:** The subject attempted to resume Phase 1 of a multi-tool workflow migration (Claude Code → Hermes Agent) using a Gemini-series language model. The session was characterized by three intersecting failure modes: (1) context window saturation with non-functional compression, (2) model execution pattern that terminated after each atomic step requiring continuous re-prompting, and (3) cross-session context persistence failure that erased prior session state and required complete re-establishment of task parameters.

**Clinical observation:** The subject presented with escalating frustration tracking alongside a specific cognitive cascade pattern. Each model hand-back (occurring after individual micro-steps rather than task-complete boundaries) triggered a full re-orientation cycle: situational assessment → task-state reconstruction → decision on continuation → prompt formulation. For an ADHD cognitive profile with high working memory load sensitivity (calibrated at 0.80/1.0), this cycle imposes a disproportionate initiation tax — the subject must repeatedly cross the conception-to-execution boundary (the gap between understanding what needs to happen and initiating the action) without the tool absorbing any of that overhead.

By the fourth cycle, the subject's frustration had generalized from the specific interaction to self-directed doubt about the viability of the entire migration strategy. This generalization pattern — concrete failure → tool assessment → self-assessment — is characteristic of ADHD rejection sensitivity and is a known risk multiplier in tool-adoption contexts.

The subject reported the model "contemplating and cogitating for 14 minutes with no indication of progress." This invisible processing period without progress markers constitutes a feedback vacuum — the absence of environmental feedback prevents the initiation-maintenance cycle from sustaining itself. For an ADHD cognitive profile, this is distinct from impatience; it is an executive function failure mode where uncertainty fills the gap left by absent signals.

**RQ5 relevance:** This session surfaces three structural design principles for adaptive platforms serving neurodiverse learners:

First, **the cost of context re-establishment between turns can exceed the cognitive budget for the task itself.** The turn-based interaction model (execute → return control → wait for prompt) that is standard in LLM tool interfaces is not neutral — it actively multiplies the initiation deficit across every step. For learners with ADHD, the number of turns required to complete a task is itself a cognitive cost variable, not just a procedural detail.

Second, **invisible processing time without progress markers creates uncertainty that erodes task commitment.** The subject reported significant emotional activation during the 14-minute stalled generation period. The absence of visible progress indicators during processing windows creates a gap that the brain fills with escalating doubt.

Third, **cross-session context persistence is not a convenience feature; it is the defining characteristic of a cognitive extension tool.** Clark & Chalmers' parity principle requires that an external resource be reliably available and automatically endorsed to qualify as cognitive extension. When context is lost between sessions, the tool fails the reliability criterion and reverts from Tier 3 (ambient extension) to Tier 2 (interactive tool), re-imposing the full initiation burden on the subject at every session start. The subject's specific phrasing — "I had to re-explain everything from scratch" — marks this as a boundary violation in the extended mind framework.

**Behavioral observations:** The subject attempted two compensatory strategies: (1) explicit instruction to the model to auto-execute across steps, which was acknowledged but not implemented, and (2) switching from the premium model (Sonnet 4.6) to a lower-cost alternative (Gemini), likely as a cost-management heuristic that inadvertently worsened the interaction pattern. The session was ultimately parked pending tool re-evaluation rather than abandoned — the subject maintained engagement despite significant affective friction.

**Adaptive recommendation:** The subject would benefit from a workflow configuration that: (a) suppresses turn-based hand-back for bounded task sequences, (b) provides real-time progress indicators during generation pauses, and (c) implements persistent cross-session context injection at session start. These align with the generation directives emphasizing momentum preservation, energy-pattern detection, and the emphasis on capability over friction.
