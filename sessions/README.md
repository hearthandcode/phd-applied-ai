# Sessions Directory

Date-based study session records. The primary longitudinal dataset for RQ5.

---

## Structure

```
sessions/
  README.md                         ← this file
  YYYY/
    MM/
      YYYY-MM-DD.md                 ← one file per study day (or YYYY-MM-DD-2.md for 2nd session)
  reviews/
    week-YYYY-WNN.md                ← weekly retrospectives (generated by /weekly-report)
```

## Relationship to other logs

| File | Contains | Update cadence |
|---|---|---|
| `sessions/YYYY/MM/YYYY-MM-DD.md` | Full session narrative + YAML metrics | After each session |
| `modules/MXX/audit.md` | Module-scoped session index (reference back to sessions/) | After each session in that module |
| `AUDIT_LOG.md` | Lightweight master index table (one row per session) | After each session |
| `SCORECARD.md` | Par vs. actual hours by module | After each session |

**Rule:** The session file is the source of truth. Module audit.md and AUDIT_LOG.md are indexes that reference it.

---

## Format

Each session file uses YAML frontmatter for all structured data + consistent markdown sections.

**YAML frontmatter** — use this for anything you want to query or analyze programmatically.
Never use "Subject: X" or free-form key-value lines in the body — put those in YAML instead.

**Markdown body** — narrative reflection in consistent sections so LLMs can parse and use it.

See `sessions/YYYY/06/EXAMPLE-session-template.md` for a complete example with sample data.

---

## Why YAML frontmatter (not "Subject: X" lines)

- Parseable by any language (PyYAML, js-yaml, Ruby, Go, etc.)
- LLMs understand YAML structure extremely well — it's reliable context
- Easy to query: `grep`, `yq`, pandas with `python-frontmatter`, or a simple script
- Consistent with the rest of the repo schema
- GitHub renders it as a clean metadata block
- "Subject: X" lines are fragile — a typo or missing colon breaks the format

## Scale anchors (use consistently across all logs)

These definitions are fixed. Using the same anchors every session makes the data comparable over time.

### Energy level (1–5)

| Score | Meaning |
|---|---|
| 1 | Depleted — can barely function; minimal output possible |
| 2 | Low — functioning but effortful; frequent friction |
| 3 | Moderate — baseline operational; normal day |
| 4 | Good — above baseline, engaged and capable |
| 5 | Peak — high output, energized, unusually productive |

### Focus quality (1–5)

| Score | Meaning |
|---|---|
| 1 | Scattered — cannot hold a thread; mind moves constantly |
| 2 | Fragmented — frequent drift; returning to task repeatedly |
| 3 | Adequate — on task with effort; manageable |
| 4 | Solid — sustained attention without much drift |
| 5 | Deep flow — full absorption; lost track of time |

### Cognitive load (qualitative)

| Value | Meaning |
|---|---|
| `low` | Working within comfortable capacity |
| `medium` | Noticeable mental effort; sustainable |
| `high` | Near capacity; fatiguing |
| `critical` | Over capacity; errors likely, rest needed |

### Recall difficulty (qualitative)

| Value | Meaning |
|---|---|
| `low` | Prior material readily accessible |
| `medium` | Some effort to retrieve; occasional blanks |
| `high` | Significant difficulty accessing prior learning |
| `n/a` | No recall task (meta-reflection, non-study sessions) |

## Using this data for research (RQ5)

The full session dataset, over 5+ years, becomes the primary corpus for RQ5.
Useful queries once data accumulates:

```python
import frontmatter
from pathlib import Path

sessions = [frontmatter.load(p) for p in Path("sessions").rglob("2*.md")]

# Energy distribution
energies = [s['energy_level'] for s in sessions if s.get('energy_level')]

# Hours by module
for s in sessions:
    for m in s.get('modules_active', []):
        # aggregate by module...
        pass
```

The JSONL export (for ML fine-tuning) is a future automation target via n8n or a script.
Each session file → one or more training examples depending on sections selected.
