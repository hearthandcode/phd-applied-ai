# phd-applied-ai — AI Assistant Context

This file bootstraps any AI assistant session in this repository.
Tool-agnostic: works with Claude, GPT-4o, Gemini, Cursor, Ollama, or any capable model.
Claude Code users: also read CLAUDE.md for tool-specific features and slash commands.

---

## What this project is

A self-directed doctoral-level curriculum in Applied AI and Machine Learning, designed for
a neurodiverse learner (ADHD + bipolar disorder) studying at variable pace. The curriculum
spans 67 modules across 6 phases, tied to a real startup thesis (Hearth & Code — an adaptive
AI-native learning platform).

This repo is simultaneously:
1. A curriculum the learner (Scott) studies from over 5–6 years
2. A longitudinal research dataset (session logs, energy, focus, mood — all public)
3. A research laboratory (H&C platform is the thesis artifact)
4. An open-source resource others can fork and adapt

Par hours per module range from 30–55 hours. Total: 3,265 par hours.

## Module structure (each of the 67 modules contains)

- `blog-post-seed.md` — scaffold for a Substack post

## Available interactions (prompt these to any model)

### Study session
Paste this to start a Socratic study session for any module:
> Read modules/M[XX]-*/theory.md and rubric.md. Ask me what I already know about this topic
> and my current competency level. Then guide me through the material using Socratic method —
> ask questions before giving answers. Adapt to my level. After 30 minutes, offer to log the
> session.

### Committee session (single advisor — works on any model)
1. Open `committee/<advisor-id>.md`
2. Copy the entire file content
3. Paste as the system prompt (or first message labeled "SYSTEM:")
4. Ask your question as yourself (the student)

Available advisors: `dr-chen.md` (ML Theory) | `dr-kowalski.md` (Systems) |
`dr-williams.md` (Education/Pedagogy) | `dr-okonkwo.md` (AI Ethics) | `dr-patel.md` (Applied AI)

### Oral examination (single advisor — recommended for models under 30B)
> You are [Dr. Name] from committee/<advisor>.md. I have just completed M[XX]. Conduct a
> 15-minute oral examination on this module from your domain perspective. Ask 3–4 questions.
> Be rigorous. Do not give answers.

For larger models (30B+), you can run all five advisors in one prompt:
> Convene all five committee advisors from the committee/ directory. Each examines me on M[XX]
> from their domain. Ask 5–7 total questions, rotating through advisors. Give a committee
> verdict at the end.

### Log a study session
> Update modules/M[XX]-*/audit.md with a new session entry. Today's date is [date].
> Hours studied: [X]. Competency self-assessment: [1–4]. Energy level: [high/medium/low].
> One-sentence summary of the most important thing from this session: [your note].
> Also update SCORECARD.md actual hours for this module to [cumulative total].

### Generate module content (use frontier model: Claude 3.5+, GPT-4o, Llama 3.3 70B)
See `docs/manual-workflow.md` for the full copy-paste generation prompt.

### Blog post draft
> Read modules/M[XX]-*/theory.md, audit.md, and blog-post-seed.md. Write a 1,200–2,000 word
> Substack post in a first-person field-report voice. Lead with the most surprising thing.
> Include technical depth, personal context (ADHD lens), and H&C platform connection.

## Choosing a module (for models that need guidance)

**No prerequisites:** M01–M06 (Phase 0) and M07–M18 (Phase 1) can be started in any order.
**Best first module:** M07 (Computability Theory) — strong archive coverage, foundational for
all AI reasoning work, connects directly to LLM expressiveness limits.
**If energy is low:** prefer modules with par ≤ 35 hours (M06, M10, M15, M16, M17, M18).
**Full prerequisite map:** `curriculum/prerequisites.md`

## Key conventions

- **Par scores** are aspirational, not deadlines. Over-par = went deep. Under-par = moved fast.
- **Off days are data** — a low-energy session logged in audit.md is research data for RQ5.
- **Competency levels:** 0 (unaware) | 1 (recall) | 2 (application) | 3 (analysis/critique) | 4 (synthesis/doctoral)
- **Language per module:** specified in theory.md front matter (Python / Rust / TypeScript / C++ / C#)
- **H&C connection:** every project has a connection to the Hearth & Code platform (thesis lab)

## Research questions (thesis)

1. How can adaptive AI systems model individual learner competency trajectories in real time?
2. What pedagogical frameworks are most effective when mediated by generative AI tutors?
3. Can computational models of personality, affect, and values improve motivation and outcomes?
4. What governance and ethics constraints are necessary for AI systems that adapt to learners?
5. How do ADHD and bipolar disorder shape a learner's trajectory through AI-mediated doctoral study?

RQ5 is an autoethnographic thread. Scott is both researcher and primary subject. All session
data is fully public by design.
